# <a name="debugging-testing-and-performance"></a>Depuração, teste e desempenho

[ Atualizado para apps UWP no Windows 10. Para ler artigos sobre o Windows 8.x, consulte o [arquivo](http://go.microsoft.com/fwlink/p/?linkid=619132) \]

Use o Microsoft Visual Studio para depurar e testar seu app. Para preparar seu app para o processo de certificação da Windows Store, use o Kit de Certificação de Aplicativos Windows.

| Tópico | Descrição |
|-------|-------------|
| [Implantando e depurando apps UWP](deploying-and-debugging-uwp-apps.md) | Este artigo apresenta as etapas para abordar vários destinos de depuração e implantação. |
| [Testando e depurando ferramentas para PLM (Gerenciamento do Tempo de Vida do Processo)](testing-debugging-plm.md) | Ferramentas e técnicas para depurar e testar como seu app funciona com o Gerenciamento do Tempo de Vida do Processo. |
| [Testar com o Emulador Microsoft para Windows 10 Mobile](test-with-the-emulator.md) | Simule a interação do mundo real com um dispositivo e teste os recursos de seu app usando as ferramentas incluídas no Emulador Microsoft para Windows 10 Mobile. O emulador é um app da área de trabalho que emula um dispositivo móvel que executa o Windows 10. Ele oferece um ambiente virtualizado no qual você pode depurar e testar aplicativos do Windows sem um dispositivo físico. Além disso, ele oferece um ambiente isolado para seus protótipos de apps. |
| [Testar apps Surface Hub usando o Visual Studio](test-surface-hub-apps-using-visual-studio.md) | O simulador do Visual Studio fornece um ambiente onde você pode projetar, desenvolver, depurar e testar os apps UWP (Plataforma Universal do Windows), incluindo os apps que você criou para o Microsoft Surface Hub. O simulador não usa a mesma interface de usuário que o Surface Hub, mas é útil para testar a aparência e o comportamento de seu app no tamanho de tela e resolução do Surface Hub. |
| [Teste beta](beta-testing.md) | O **teste beta** dá a você a chance de aprimorar seu app com base nos comentários de pessoas de fora de sua equipe de desenvolvimento que experimentam seu app não lançado em seus próprios dispositivos. |
| [Windows Device Portal](device-portal.md) | O Windows Device Portal permite que você configure e gerencie seu dispositivo remotamente por uma rede ou conexão USB. |
| [Kit de Certificação de Aplicativos Windows](windows-app-certification-kit.md) | Para dar a seu app maior probabilidade de ser validado pelo [Kit de Certificação de Aplicativos Windows](http://go.microsoft.com/fwlink/p/?LinkID=309666). |
| [Desempenho](performance-and-xaml-ui.md) | Os usuários esperam que seus apps mantenham a capacidade de resposta e naturalidade no uso e não usem muito a bateria. Tecnicamente, o desempenho é um requisito não funcional, mas tratar o desempenho como um recurso ajudará você atender às expectativas dos usuários. Especificar metas e mensurar são fatores importantes. Determine quais são os cenários críticos de desempenho; defina o que significa bom desempenho. Em seguida, faça medições o quanto antes e com frequência suficiente ao longo do ciclo de vida do projeto para cumprir suas metas. |
